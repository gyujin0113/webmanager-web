## 2. 수익 구조 — 3개 라인 (razor-and-blades)

> **핵심 원리:** 제작(razor)은 *고객 획득 도구*, 구독(blade)이 *본진 매출*, 기능개발은 *옵셔널 큰 수표*.
> 시간·현금을 **구독 MRR 쌓기**에 몰빵하고, 제작은 비용 최소화, 기능개발은 시간 상한 두고 기회만.

### 라인 1 — 제작/온보딩 (CAC 도구, 매출 아님)
표준스택 재구축(04 §3)은 *고객용 상품*이 아니라 **마진엔진을 돌리기 위한 내부 온보딩**. 그래서 "제작비를 판다"는 프레임 자체가 틀렸다. 3가지 운용 모드:

| 모드 | 누구에게 | 가격 | 비고 |
|---|---|---|---|
| **A. 정부 바우처 충당** | 자격되는 중소기업 | 고객 무료 / **정부가 우리에 지급** | 최선. CAC를 정부가 댐. 공급기업 등록 ⚠️선검증 |
| **B. 무료 새단장** | **초기 10명** (증거·로고 확보용) | 0원 | 콜드 전환율 극대화. before/after + 후기 수확 |
| **C. 셋업비** | 증거 쌓인 뒤 신규 | **49~99만** (시세 400만의 1/4~1/8 앵커) | 타이어킥커 필터 + CAC 일부 회수. *"제작비"* 아니라 *"셋업비"*로 프레이밍 |

→ 결론: **"새 홈페이지 사세요"가 아니라 "무료(또는 정부지원)로 새단장 + 그때부터 월 구독으로 관리".**

### 라인 2 — 구독 (본진, 정액 무제한 / 상세는 02)
복잡도·needs로 티어, **횟수로 가르지 않음**(건당 마찰 = status quo 회귀). 시장 천장($199~997) 대비 한참 아래라 인상 여지 큼.

| 티어 | 월 | 포함 | 대상 |
|---|---|---|---|
| **Founding(친구가)** | 5만 (평생고정) | 콘텐츠 무제한 | 기존 2고객. 절대 안 올림 |
| **Basic** | 7~9.9만 | 콘텐츠 무제한 + 위젯 1회셋업 + 월점검 | 소규모 기술회사 회사소개 사이트 (ICP) |
| **Standard** | 12~15만 | Basic + 호스팅·도메인 관리 + 기본 SEO + 정기 업데이트(공지·블로그 운영) + 월 리포트 | 사이트가 영업에 중요한 곳 |
| **Pro** | 25만 | Standard + 멀티 사이트 / 우선 SLA / 성과 리포트 | 다지점·여러 브랜드 |

### 라인 3 — 별도 프로젝트 (옵셔널 1회성 수표)
구독 우산 **밖**. 04 §4 티어③·④에 해당. 시간 상한 두고 기회만.

| 종류 | 04 분류 | 가격대 | 처리 |
|---|---|---|---|
| 새 페이지·섹션 틀, 리디자인, 대량 콘텐츠 이관, 본격 SEO, 이미지 제작·보정, 카피라이팅 | 티어③ | **수십만 견적** | 먼저 견적 후 진행 (업셀) |
| 로그인·결제·회원DB·거래처 포털·자체 백엔드 (버킷②) | 티어④/별도상품 | **수백~1,000만+ 프로젝트** | 구독서 분리한 *별도 프로젝트*로만. 또는 거절/파트너 소개. AI보조 개발 가능하나 시간 상한 필수 |

> ⚠️ ICP가 *기술회사*라 "거래처 로그인 만들어줘" 요청 확률이 일반 가게보다 높음 → **경계를 온보딩 때 못박을 것.** 자체백엔드 1명이 정적 10명 마진을 까먹는다.

## 3. 운영 모델 — "Productized Solo + AI Factory"

파트타임(주 10~20h) 1인 + AI라는 제약이 모델을 강제한다: **극단적으로 productize.** 매번 새로 짜면 1인이 못 버틴다.

**공장 관점 (고객 1명 = 동일 파이프라인 통과):**
```
콜드/소개 인입 → (자격: 버킷① 회사소개형인가?) → 표준스택으로 클론·새단장(온보딩)
  → 구독 시작(자동이체) → 이후 수정요청은 [텍스트→AI 적용→규진 사람게이트→배포→git revert 롤백]
  → 만족 → 추천 + 푸터 노출(인바운드 씨앗) → 별도요청 들어오면 라인3로 분기
```

**1인이 시간을 쓰는 곳 vs AI/시스템에 맡기는 곳:**
| 사람(규진) | AI/시스템 |
|---|---|
| 영업(콜드·소개), 사람 게이트(diff 승인), 별도 프로젝트, 관계 | 콘텐츠 수정 적용, 미리보기 빌드, 에셋 R2 파이프라인, 프로스펙팅 점수화 |

**재사용으로 한계비용을 0으로:** 한 니치(소규모 기술/제조)로 좁힐수록 표준스택 1개·위젯셋업·AI 편집패턴이 똑같아져 **재사용 복리** → 마진엔진이 굳음. (= WaaS 성공공식 "니치+표준화"와 동일)

**매출 믹스 목표:** 구독 MRR = 안정 기반(쌓기). 제작 = 0에 수렴(무료/바우처). 별도 = 기회성 현금(시간 상한). **"20x 컴퍼니"** = 고객 늘어도 시간 거의 안 늘어남.

## 4. 가장 가능성 높은 방향 (권고)

> **"AI-운영형 회사소개 사이트 구독" = Managed-Brochure WaaS + AI 마진엔진.**
> 소규모 기술/제조 회사의 방치된 회사소개 사이트를 → 무료(또는 정부지원) 새단장으로 인입 →
> 월 7~15만 정액 무제한 구독으로 운영 → 수정은 AI가 치고 규진이 게이트 → 기능개발은 별도 프로젝트.

**왜 이 방향인가 (Why):**
1. **메가트렌드 정합** — WaaS·구독화·니치특화가 2026 성공공식. 역풍 아니라 순풍 위에 탐.
2. **AI가 진짜 비용우위** — "무제한"을 사람이 하면 월 $1,000+, AI면 한계비용≈0. 이 격차가 곧 마진이자 가격 무기.
3. **에이전시는 구조상 못 따라옴** — 사람이 무제한 치면 적자. 우린 AI라 흑자. 노코드 셀프툴은 우리 ICP가 안 씀.
4. **천장이 열려 있음** — 시세 대비 한참 낮게 시작 → 증거·신뢰 쌓고 인상.
5. **CAC를 낮출 두 레버** — 제작 무료화(razor) + 정부 바우처(있으면 CAC를 정부가 댐).

**해자·타이밍 (정직):**
- AI 에이전트 *기술*은 해자가 아니다(누구나 Agent SDK로 배선 가능). 진짜 해자 = ①한 니치에 맞춘 *운영 시스템*(표준스택+범위규율+사람게이트) ②고객 관계·유지율 ③**니치 플라이휠을 경쟁자보다 먼저 도는 속도.**
- 인입 도구가 상품화될수록 미래 진입장벽↓ → **지금 1년이 골든타임.**
