## PART A. 서비스 카탈로그 — 3+1 분할

fork9는 7개 카테고리를 나열하지만 "어디까지 포함이냐"는 흐릿하다. 너는 **포함/별도/안함**을 못박아
그 흐릿함 자체를 차별점으로 만든다.

### ✅ 구독에 포함 — "무제한" (콘텐츠 수정)
- 텍스트 수정 (소개문, 설명, 공지, 이벤트 문구)
- 영업시간 / 휴무일 변경
- 가격표 / 메뉴 / 시술·상품 목록 갱신
- 이미지 교체 (고객 제공 이미지 기준)
- 연락처 / 주소 / 지도 / SNS·예약 링크 갱신
- 배너 교체, 섹션 노출 순서 변경
> 이게 "무제한"의 진짜 범위. 자유 텍스트로 카톡 주면 당일 반영.

### 🔁 구독에 포함 — 운영 관리 (정기, 무제한 아님)
- 도메인 · SSL 인증서 갱신 관리
- 정기 데이터 백업
- 가동 모니터링 (사이트 다운 시 알림 + 복구)
- 월 1회 상태 점검 + 리포트

### 💬 별도 견적 — 업셀 (한 번에 끝나는 작업)
- 신규 페이지 추가
- 리디자인 / 레이아웃·구조 변경
- 신규 기능 (온라인 예약, 결제, 문의폼, 챗봇/알림톡 연동)
- 본격 SEO 작업 (검색 상위노출 프로젝트)
- 사진 촬영 / 이미지 제작
> "수정 ≠ 개발". 이 경계가 '무제한이라며 이것도 안 돼?' 분쟁을 막고 추가매출 통로를 연다.

### 🚫 안 함 — 정직하게 (신뢰는 여기서 나온다)
- 광고 집행 대행 / 콘텐츠 기획·카피라이팅 (→ 필요 시 파트너 소개)
- 대규모 서버 인프라 / 트래픽 폭주 대응
> 못 하는 걸 못 한다고 적는 게 28년 업력보다 강한 신뢰장치다. 1인이라 더 그렇다.

## PART B. 랜딩페이지 콘텐츠 청사진 (순서대로)

> 핵심 원리: **랜딩 카피 = 제품 스펙.** "카톡으로 수정 요청 → 당일 반영"이라 약속하면,
> 그게 곧 MVP가 구현해야 할 명세다. 마케팅과 구현은 따로가 아니다. 정직하게 쓰면 백로그가 된다.

1. **히어로** — 한 줄 가치 + CTA
   - 헤드라인 예: "홈페이지, 이제 카톡으로 고치세요"
   - 서브: "수정은 무제한 · 처리는 당일 · 비용은 월 정액. 만들고 끝이 아니라 평생 관리."
   - CTA 버튼: "내 홈페이지 무료 진단받기"

2. **문제 공감 (Before)** — 고객의 현재 고통을 거울로
   - "영업시간 바뀌었는데 사이트는 한 달째 옛날 그대로"
   - "수정 한 번 하려고 업체에 메일 쓰고, 며칠 기다리고, 또 추가비용"
   - "Wix로 직접? 만들다 포기"

3. **솔루션 (After)** — 3단계 그림
   - ① 카톡으로 "영업시간 10시로 바꿔줘" → ② 우리가 반영+미리보기 → ③ 당일 라이브 반영

4. **서비스 카탈로그 (PART A의 3+1 분할 테이블)** ← 네가 원한 카탈로그. 신뢰의 심장.

5. **가격** — 정액 무제한, 티어
   - Basic 7~9.9만 / Standard 12~15만 (호스팅·SEO기본·리포트 포함). "추가금 없음" 강조.

6. **차별점** — 3자 비교표
   - vs 셀프(Wix·아임웹: 직접 해야 함) / vs 에이전시(느리고 비쌈) / **3&D: 무제한+당일+정액**

7. **신뢰 요소** — 너만의 방식
   - 실제 사례: 미용실, 진엽 형 회사 — before/after 스크린샷 + 한 줄 후기(받아둬라)
   - fork9는 대기업 레퍼런스로, 너는 "동네 사장님이 진짜 쓴다"는 친근한 신뢰로.

8. **FAQ** — fork9 벤치마크 + 너 특화
   - "타사가 만든 홈페이지도 관리되나요?"
   - "무제한이면 진짜 횟수 제한 없나요?" / "어디까지가 무제한인가요?" (경계 재확인)
   - "최소 약정 기간 있나요?" / "내 사이트 소스코드는 제가 소유하나요?"

9. **CTA 반복** — 진입장벽 최저 행동
   - "카톡으로 1분 상담" 또는 "무료 진단" (구매 아닌 대화부터)

10. **푸터** — 사업자 정보(3&D), 연락처, 카톡채널, 도메인 www 라우팅 점검(현재 www 연결 거부)

## 마케팅 ↔ 구현 연결 (네 질문의 핵심)
- 랜딩의 **3번(솔루션 흐름) + 4번(카탈로그)이 곧 MVP 스펙**이다.
- "카톡 → 당일 반영" 약속은 MVP(A+C: 카톡 인입 + 사람게이트)로 *지금도* 정직하게 지킬 수 있다.
  (AI 자동화 전이어도 네가 수동으로 당일 처리하면 약속 이행 가능 → 과대광고 아님)
- 즉 **랜딩을 먼저 정직하게 쓰면, 그게 그대로 개발 백로그가 된다.** 둘은 한 몸.
