## 1. 경쟁사 시세 리서치 결론 (2026, 한국 소상공인)

> ⚠️ **신뢰도 주의**: 딥리서치 검증 단계에서 API 레이트리밋으로 검증봇 다수가 "기권" 처리됨.
> 아래 ✅는 3-0 확정, ⚠️는 1차 출처는 있으나 미검증(참고치). 숫자는 환각 위험 구간이라 표기 구분.

| 채널 | 제작비(1회성) | 유지보수(월) | "무제한"의 실제 | 신뢰도 |
|---|---|---|---|---|
| 종합 에이전시 (fork9) | 수백만~ | **10만(STANDARD)~20만(DELUXE,+SEO)** | 콘텐츠 교체만. 기능은 건당 | ✅3-0 |
| 에이전시 AS (DWEBS) | — | 횟수티어: 월 2회/5회/무제한 | 무료수정=기본정보·텍스트/이미지 1:1 교체뿐. 페이지추가 9만, 프로그램수정 5~30만+ | ✅3-0 |
| 1인/마켓 (숨고) | — | **월정액 거의 없음. 건당 평균 30만**(5만~300만) | 건당 구조 | ✅3-0 |
| 노코드 SaaS (아임웹) | 셀프 | **1.3만~2.4만/월**(도구 사용료) | "무제한"=페이지/상품/트래픽 무제한, **수정 대행 없음** | ✅3-0 |
| 1인 프리랜서 구독 (크몽 gig) | ~29만 | 9.9천 / 2.9만 / 3.9만 / 5.9만 | "무제한 수정" 표방하나 기능 제외 | ⚠️미검증 |
| 분쟁 사례 (소비자원) | — | 제작+1년 유지보수 540만/년(≈45만/월) | 범위·완성 모호로 분쟁 | ⚠️미검증 |

**핵심 발견 3가지**
1. **"무제한 수정"은 시장 전체가 사실상 콘텐츠 교체만 의미.** 어디든 ①횟수 제한 티어 or ②기능은 건당
   추가금. **진짜 횟수무제한 + 추가금0은 시장에 거의 없음.**
2. **"유지보수 범위 모호" 불만은 실재** (✅2-1, fork9가 자사 마케팅에서 경쟁사 불만으로 정면 공략 +
   복수 블로그 교차). → 03의 "투명한 범위" 무기가 시장 공백을 정확히 찌름.
3. **시장은 바벨 구조.** 노코드 1~2만(셀프, 내가 다 함) ↔ 에이전시 10~20만(대행하지만 "무제한"=
   콘텐츠만+추가금). **그 중간 "대행 + 진짜 무제한 + 추가금0 + 투명"이 비어 있음 = 3&D 자리.**

**가격 시사점**
- **5만원 = "크몽 1인 프리랜서" 가격대.** 공개 5만이면 시장이 3&D를 *저가 프리랜서*로 인식 → 가격경쟁.
- **7~9.9만 = "1인인데 에이전시급 대행" 포지션.** 02의 신규가 판단을 시세가 뒷받침.
- **상위 티어(스탠다드 15만/프로 25만)는 이미 시세 정합** — 안 건드려도 됨.
- **진짜 해자는 가격이 아니라**: 진짜 무제한 + 추가금0 + 범위투명 + (AI라서) 당일. 에이전시는
  마진구조상 못 따라옴(사람이 무제한 치면 적자). AI 마진엔진이 이걸 가능케 함.
- **결정**: 친구 2명 5만 평생고정 유지 / 공개 신규가 바닥은 7만(9.9만 권장). **최종 숫자는 미확정(보류).**

## 2. ICP & 3버킷 (받을 고객의 경계)

전체 마진은 **"AI가 구조화 데이터를 한계비용≈0으로 편집"**에서 나온다. 동적 백엔드는 이걸 깬다
(①로직이라 AI가 안전편집 불가 = 위험한 auto-deploy ②DB·인증·결제를 1인이 떠안음 = 레버리지 붕괴).
하지만 소상공인 "동적" 욕구의 ~90%는 외부 SaaS 위젯 임베드로 흡수 가능 → 백엔드를 안 가져도 충족.

| 버킷 | 내용 | 처리 |
|---|---|---|
| **① 정적 + 외부위젯 흡수** (ICP 99%) | 미용실·식당·동네병원·소개/포트폴리오. 예약=네이버예약/캘린들리, 폼=Web3Forms/Tally, 지도=임베드 | ✅ **받는다.** 위젯은 1회 셋업, 콘텐츠는 무제한 |
| **② 자체 백엔드 필수** | 자체 재고/결제 쇼핑몰, 자체 회원/로그인, 고volume 자체 게시판 | 🚫 **지금은 보류.** (또는 그 기능만 기존 카페24/아임웹에 남기는 하이브리드) |
| **③ 나중에** | 규모 생기면 ②를 무제한 구독과 분리된 *프로젝트성 별도 상품*으로 | 🔜 확장 시 |

**왜 지금 ①로 좁히나**: 가설("AI 마진엔진으로 무제한 수익화")이 ②에선 안 돈다. ②를 섞으면 검증
신호가 오염. **가진 고객 2곳이 정확히 ①.** ①로 모델 증명 후 ②③으로 TAM 확장. (영구 배제 아님, YAGNI.)

**온보딩 판정 기준 (스택이 아니라 *실사용 동적*으로 판정)**
> ❗ "워드프레스로 만들었다 ≠ 동적이다." 소상공인 WP의 다수는 기능적으론 그냥 5페이지 소개사이트.
> 던질 질문: **"사장님이 *직접 손대는* 기능이 뭐예요? 예약 받아요? 회원 있어요? 결제해요?"**
> - 살아있는 동적 없음(겉만 WP) → 버킷① 받음.
> - 진짜 동적 살아있음(우커머스 매출·회원DB) → 버킷② 보류.

## 3. 표준 스택 아키텍처 (온보딩 시 재구축)

**결정: 고객 사이트를 표준 스택으로 클론·재구축한다** (as-is로 받지 않음).
이유: AI가 *코드*가 아니라 *구조화된 데이터 값*을 고치게 되어 위험≈0 + 한계비용≈0. 이게 해자.

> **표준 스택 = Next.js 정적 + 콘텐츠는 파일(MDX, git) + 에셋은 외부 스토리지(R2) + 동적은 외부 위젯 흡수 + 1수정=1커밋 롤백.**

### 3.1 웹 3세대 맥락 (왜 이렇게 가나)
- **1세대 정적 HTML**: 개발자만 수정 가능 → 고객 고통의 원형.
- **2세대 PHP/워드프레스(2003)·그누보드·카페24**: 비개발자가 관리자(CMS)에서 직접 관리. 한국
  소상공인 사이트의 다수. webmanager-web "4유형 고민"(연락두절·인질·추가금·고정비)이 이 세대 유산.
- **3세대 React(2013)/Next.js(2016)**: 정적 생성 + 동적은 외부 API/서버리스 분리(JAMstack).
- 3&D 모델 = 2세대 사이트를 3세대 정적으로 옮김. **잃는 것 = 고객 셀프 wp-admin** — 그런데 ICP
  고객은 어차피 그걸 못 쓰던 사람들 → "관리자 씨름 → 텍스트 한 줄 요청"으로 *대체*. 이것이 가치제안.

### 3.2 콘텐츠/에셋 관리 (핵심 제약: git은 바이너리에 약하다)
| 무엇 | 어디에 | 이유 |
|---|---|---|
| **텍스트 콘텐츠** (글·공지·문구 MDX) | **Git 레포** | 작고 diff 잘 됨. 버전관리가 *기능* → 롤백 공짜 |
| **이미지·영상·PDF** | **Cloudflare R2 / 이미지 CDN** (git 밖) | 바이너리는 diff 안 돼 히스토리 영구 부풂. 빼야 레포 가벼움 |
| 레포엔 **에셋 URL 참조만** | Git | 이미지 100개 = git엔 짧은 참조 100줄, 실물은 R2 |

- 블로그/공지 = "글 1개 = MDX 파일 1개", Next가 라우트 자동 생성. 저빈도(고객당 월 1회)라 빌드타임 OK.
- **에셋 업로드 흐름**: 고객 이미지 업로드 → R2 저장(URL 발급) → AI가 URL을 콘텐츠 파일에 참조 삽입 +
  커밋 → 바이너리는 git에 안 들어감. 텍스트는 깨끗한 diff로 사람게이트·롤백 유지.

### 3.3 Phase 2 자동화 흐름 (= 01의 A+C MVP)
```
고객이 [수정문구 + 첨부파일]을 (홈페이지 포털 또는 카톡으로) 업로드
  → 에셋은 R2로, 요청 텍스트는 AI 에이전트로
  → AI 에이전트가 [그 고객 사이트 레포 + 요청]으로 수정 적용 (텍스트 직접 / 이미지 URL 참조)
  → 미리보기 빌드 (변경 전후 비교)  ← Cloudflare Pages 브랜치 프리뷰 활용 가능
  → [사람 게이트] 규진이 폰에서 diff/미리보기 승인
  → 승인 시 배포 (git commit → push → 자동배포)
  → 롤백 = git revert (1수정 = 1커밋)
```
- 엔진 후보: Claude Code / Claude Agent SDK (레포 편집 → 빌드 → diff 생성 = 정확히 이 일).
- 게이트는 2단(규진의 안전 게이트 + 고객의 "이걸로 바꿔" 승인)일 수 있으나, 초기엔 1단으로 합쳐
  운영(규진이 검토 후 고객에게 미리보기 전달).

## 4. 수정범위 — 4티어 + 판정 규칙 (시장 #1 불만 역공략)

### 4.0 판정 규칙 (열거보다 중요 — 새 케이스도 이걸로 판정)
> **무제한 = "이미 있는 것의 *값*을 바꾸는 것"(편집). 별도 = "없던 것을 *만들거나* 구조·로직을 바꾸는 것"(제작·개발).**
>
> **"무제한"은 *요청 횟수*에 거는 약속이지 *한 건의 작업량*에 거는 게 아니다.** 요청 횟수는 무제한
> (건당 과금 금지). 단 한 건이 사실상 프로젝트(대량·창작·구조)면 → 횟수에서 빼고 "별도 작업"으로 재분류.

### 4.1 4티어
| 티어 | 내용 | 처리 |
|---|---|---|
| **① 무제한 편집** | 텍스트 수정/입력, 가격·시간·연락처·주소, 이미지 *교체*, 공지·블로그 글 *추가*(기존 틀), 항목 on/off·순서, 배너 교체, 지정된 값 변경("버튼 파랑→빨강") | ✅ 무료·무제한, 영업일 48h 내 |
| **② 회색지대** | 주관적 디자인 취향("더 예쁘게"), 가벼운 레이아웃 미세조정 | 🟡 건당 **2회 수정까지 포함**, 이후 '디자인 변경' 견적 |
| **③ 별도 견적(업셀)** | 새 페이지·새 섹션 *틀*, **페이지 라우팅/구조 변경**, 리디자인, 새 기능(예약·결제·폼·지도=위젯 셋업), 본격 SEO, 이미지 *제작·보정*, 카피라이팅, 대량 콘텐츠 이관 | 💬 먼저 견적 후 진행 |
| **④ 안 함(정직)** | 광고 집행 대행, 자체 백엔드 동적(버킷②), 대규모 인프라, 저작권 없는 자료 게시 | 🚫 거절 / 파트너 소개 |

### 4.2 분쟁 유발 상황 × 예방 정책
| 분쟁 트리거 | 무제한인 척하지만 사실은 | 예방 정책 |
|---|---|---|
| "텍스트 고쳐줘" → 전체 카피 다시 써줘 | 수정 vs 창작(카피라이팅) | 문구 *수정·입력*은 무제한 / *대신 써주는* 글은 별도 |
| "이미지 교체" → 포스터 제작·누끼·보정 | 교체 vs 제작 | 고객 제공 이미지 *교체* 무제한 / 이미지 *제작·합성·보정* 디자인 별도 |
| "공지 올려줘" → 새 이벤트 페이지 | 글 1개 vs 새 *틀* | 기존 틀 안의 글 무제한 / 새 *페이지 틀* 별도 |
| "버튼 위로/색 바꿔" → "더 고급지게" 무한루프 | 지정 값 vs 주관 취향 | 지정 변경 무제한 / 주관 디자인 2회 후 견적 |
| 매달 제품 100개 일괄 갱신 | 편집이나 노동 폭탄 | 한 건이 대량이면 '정기 업데이트 작업'으로 재분류·일정 협의 |
| "당일이라며 왜 안 돼" | SLA 기대 불일치 | 영업일 48h 내, *고객이 자료 준 시점*부터 카운트 |
| 고객이 문구·이미지 안 줌 | 책임 소재 | 고객 제공물 대기 시간은 SLA 제외 |
| 예약 SaaS·도메인 장애로 깨짐 | 외부 요인 | 외부 위젯/제3자 변경발 문제는 '별도 복구'(모니터링은 포함) |
| "검색 1등 만들어줘" | 성과형 | 검색 등록·기본 SEO 포함 / 순위 보장·본격 SEO 별도 |
| 저작권 없는 이미지·개인정보 게시 | 법무 리스크 | 거절 사유 명시 |

### 4.3 약관·온보딩에 *그대로 박을* 문장
1. "무제한은 요청 **횟수**에 대한 약속입니다. 한 건의 요청이 새 페이지·새 기능·대량 제작에 해당하면 **별도 작업으로 분류**해 먼저 견적을 안내합니다."
